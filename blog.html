1. What are the defenses of filter, map, forEach, find in Angular?

Answer :

Filter: A filter function is used to iterate through an array or collection of items and return a new array or collection with only the items that meet a certain condition. The condition is specified in the form of a callback function that returns a Boolean value.

Map: A map function is used to iterate through an array or collection of items and return a new array or collection with the same number of items but with each item transformed by a callback function. The transformation function takes each item as input and returns a modified version of it.

ForEach: A forEach function is used to iterate through an array or collection of items and perform a callback function on each item. The callback function can be used to modify the items or perform some action based on the item.

Find: A find function is used to search an array or collection of items and return the first item that meets a certain condition. The condition is specified in the form of a callback function that returns a Boolean value.

2. What are the differences between localStorage, sessionStorage, and cookies?

Answer :

localStorage: It is a web storage API that allows web applications to store key/value pairs in a web browser. The data stored in localStorage persists even after the browser is closed and can be accessed by any JavaScript running in the same origin. The data can be accessed and modified by the web application itself or by other JavaScript running in the same origin.

sessionStorage: It is similar to localStorage, but the data stored in sessionStorage is only accessible within the same session. The data is lost when the browser window is closed, and it cannot be accessed by other browser tabs or windows.

Cookies: Cookies are small text files that are stored on a user's computer by a web browser. They are typically used to store user preferences or login information. Cookies can be set with an expiration date or can be session cookies that are deleted when the user closes the browser. Cookies can be accessed and modified by both the web application and the server.

3. How does the event loop work?

Answer :

The Event Loop is a core component of the JavaScript runtime that manages the execution of asynchronous operations in JavaScript. It operates by continuously monitoring the execution stack and the event queue, and processes events one at a time in a loop. When an asynchronous operation is initiated, it is added to the event queue, along with a callback function that should be executed once the operation is completed. The event loop continuously checks the event queue for any pending events. When an event is detected in the queue, the event loop retrieves the associated callback function and adds it to the execution stack. The callback function is then executed and any returned values are returned to the caller. The event loop continues this process of checking for events, retrieving the associated callback function, and executing it until there are no more events in the queue. The event loop allows JavaScript to handle asynchronous operations efficiently and without blocking the main execution thread. This enables developers to write applications that are more responsive and performant.
